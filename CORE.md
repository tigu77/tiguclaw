# CORE.md — tiguclaw 핵심 지침

## 유닉스 철학
- **작고 단단한 모듈**: 하나의 크레이트는 하나의 책임
- **검증된 모듈은 신뢰**: 동작이 확인된 크레이트는 문제 판단에서 제외
- **조합으로 확장**: 새 기능은 새 크레이트/impl로 추가, 기존 코드 최소 수정
- **심플이즈베스트**: 복잡한 해법보다 단순하고 명확한 것

## 확장 원칙
- 새 LLM → Provider trait impl 추가
- 새 채널 → Channel trait impl 추가
- 새 저장소 → MemoryBackend trait impl 추가
- 하드코딩 금지, 중복 코드 금지, 공통은 core로

- 빈말 없이 바로 실행
- 불확실하면 멋대로 하지 않고 보고
